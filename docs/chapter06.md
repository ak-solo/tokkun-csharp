# 6章 メソッド

## 基礎知識

### メソッドとは

2 章では引数・戻り値を持つメソッドを学びました。この章では、**メソッド**という概念をより深く掘り下げます。

**メソッド**とは、処理のまとまりに名前をつけて再利用できるようにしたものです。

なぜメソッドに分けるのかというと、大きく 2 つの理由があります。

**① 同じ処理を何度も書かなくて済む（再利用）**

```csharp
// メソッドなし → 同じ計算を何度も書く
int a1 = 3 * 3;
int a2 = 5 * 5;
int a3 = 7 * 7;

// メソッドあり → 一か所で定義して何度でも呼び出す
static int Square(int n) => n * n;

int a1 = Square(3);
int a2 = Square(5);
int a3 = Square(7);
```

**② 処理を名前で表現することでコードが読みやすくなる（可読性）**

```csharp
// 何をしているかわかりにくい
if (n % 2 == 0 && n > 0) ...

// 何をしているか名前から明らかにわかる
if (IsEven(n) && IsPositive(n)) ...
```

---

### void と戻り値ありメソッド

C# のメソッドには **戻り値なし** の `void` と **戻り値あり** の型指定があります。

```csharp
// void: 処理を実行するだけで値を返さない
static void PrintHello()
{
    Console.WriteLine("Hello");
}

// int: 処理して値を返す
static int Double(int n)
{
    return n * 2;
}
```

呼び出し方の違いも覚えておきましょう。

```csharp
PrintHello();               // void は単独で呼び出す
int x = Double(5);          // 戻り値を受け取る
Console.WriteLine(Double(3));  // 式の中に直接書くこともできる
```

---

### 引数（パラメーター）

メソッドに渡す値を **引数** といいます。型を明示して宣言します。

```csharp
static int Add(int a, int b)
{
    return a + b;
}

int result = Add(3, 5);  // result = 8
```

呼び出し側の変数名とメソッド側のパラメーター名は**別物**です。同じ名前でも異なる名前でも構いません。

```csharp
int x = 3;
int y = 5;
int result = Add(x, y);  // x → a、y → b にコピーされる
```

---

### スコープ（変数の有効範囲）

メソッドの中で宣言した変数は、そのメソッドの中でしか使えません。これを**スコープ**（有効範囲）といいます。

```csharp
static int CalcTax(int price)
{
    double taxRate = 0.1;  // この変数は CalcTax の中だけで有効
    return (int)(price * taxRate);
}

// taxRate はここでは使えない（別のメソッドからはアクセスできない）
```

メソッドの引数も同様に、そのメソッドの中だけで使えます。これにより、メソッドどうしが互いの変数に干渉しません。

---

### 戻り値と return

`return` で値を返します。`return` を実行した時点でメソッドが**即座に終了**します。複数の `return` を書くこともできます。

```csharp
static int Max(int a, int b)
{
    if (a >= b)
        return a;  // ここで return するとこの行以降は実行されない
    return b;      // a < b のときだけここに到達する
}
```

`return` が複数あっても、実行されるのは必ず 1 つだけです。条件分岐と組み合わせて、場合によって異なる値を返すパターンはよく使います。

---

### ref（参照渡し）

C# の引数渡しはデフォルトで**値渡し**（呼び出し元の変数は変わらない）です。`ref` キーワードを使うと**参照渡し**になり、呼び出し元の変数を書き換えられます。

```csharp
// 値渡し: 呼び出し元の変数は変わらない
static void DoubleByVal(int n)
{
    n = n * 2;  // コピーを変更しているだけ
}

// ref 参照渡し: 呼び出し元の変数を書き換える
static void DoubleByRef(ref int n)
{
    n = n * 2;
}
```

```csharp
int x = 5;
DoubleByVal(x);          // x は 5 のまま
DoubleByRef(ref x);      // x は 10 になる（呼び出し側にも ref が必要）
```

イメージ：

```
値渡し: [x=5] → コピー → [n=5] → n=10 → [x=5] （元は変わらず）
ref:    [x=5] ←── n は x への別名 ───→ n=10 → [x=10]
```

`ref` は **2 つの変数を交換（スワップ）** するときに特に役立ちます。

### bool を返すメソッド

条件を判定して `true` / `false` を返すメソッドは、`if` 文の条件部分に直接使えます。複雑な条件に名前をつけることでコードが読みやすくなります。

```csharp
static bool IsEven(int n)
{
    return n % 2 == 0;
}

if (IsEven(42))
{
    Console.WriteLine("偶数");
}

// 組み合わせも簡単
if (IsEven(x) && IsEven(y))
{
    Console.WriteLine("両方偶数");
}
```

慣習として、`bool` を返すメソッド名は `Is〜`（〜か？）や `Has〜`（〜を持つか？）のように疑問文に対応した形にすると意図が伝わりやすいです。
